# Reflexion del prototipo medio nivel

Durante las pruebas del prototipo medio nivel se reviso que tan clara era la navegacion, la seleccion de productos y la pantalla final de confirmacion.

## Prueba 1

Se le pidio al usuario navegar por la aplicacion desde la pantalla de inicio hasta la pantalla de confirmacion. El objetivo fue comprobar que tan intuitivo era el flujo general.

Los usuarios pudieron avanzar sin muchos problemas, pero comentaron que el diseno podia verse mas moderno y parecido a aplicaciones como Rappi o Uber Eats.

## Prueba 2

Se le pidio al usuario seleccionar un chocolate caliente desde el menu. Esta prueba ayudo a identificar como debian funcionar las areas presionables de cada producto.

El hallazgo principal fue que algunos usuarios no presionaban directamente el boton de opcion, sino el nombre del producto. Por eso, se propone que el area presionable incluya tanto el boton como el texto del producto.

## Prueba 3

Se le pidio al usuario elegir el boton que le pareciera mas intuitivo en la pantalla final, entre Tracker, Otro pedido y el icono de menu principal.

La mayoria eligio Tracker. Casi nadie uso el icono de menu principal ni el boton de Otro pedido. A partir de esto, se propone agregar una opcion mas clara para volver al menu principal, en lugar de depender solo de un icono de casa.

## Cambios propuestos

- Modernizar el diseno visual del prototipo.
- Ampliar el area presionable de los productos en el menu.
- Hacer mas clara la salida hacia el menu principal desde la pantalla final.
- Revisar la jerarquia de botones en la pantalla final para que las acciones importantes sean mas evidentes.
