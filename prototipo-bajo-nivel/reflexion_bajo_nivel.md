# Reflexion del prototipo bajo nivel

Durante las pruebas del prototipo bajo nivel se reviso que tan claro era el flujo para crear una orden, consultar ordenes pasadas, seleccionar sucursal y confirmar el pedido.

## Ordenes pasadas y nueva orden

Uno de los principales hallazgos fue que la pantalla de ordenes pasadas y la opcion de hacer una nueva orden no fueron tan claras para los usuarios. En algunos casos, se omitia el boton de ordenar dentro de la pantalla de ordenes pasadas porque no se entendia bien que desde ahi se podia iniciar o repetir una orden.

Se propone cambiar la ubicacion de esta opcion o redisenar esa pantalla para que sea mas facil diferenciar entre consultar pedidos anteriores e iniciar una nueva orden.

## Cantidad de productos

Tambien se observo que seria util permitir agregar mas de un producto dentro de una misma orden. En lugar de pedir solo un cafe por pedido, la aplicacion deberia permitir seleccionar la cantidad deseada, por ejemplo pedir varios cafes en una sola orden.

Este cambio haria el flujo mas practico para usuarios que quieren pedir para mas de una persona o comprar varios productos antes de entrar a clase.

## Seleccion de sucursal

La seleccion de sucursal no fue completamente comoda para los usuarios. La funcion sigue siendo importante, ya que ayuda a definir donde se recogera el pedido, pero podria funcionar mejor si aparece en otro momento del flujo.

Cambiar la ubicacion de esta pantalla podria hacer que el proceso sea mas natural y menos confuso, especialmente antes de confirmar la hora de entrega.

## Cambios propuestos

- Redisenar la pantalla de ordenes pasadas para que sea mas clara.
- Hacer mas visible la opcion de hacer una nueva orden.
- Evitar que el boton de ordenar pase desapercibido.
- Permitir elegir la cantidad de cafes o productos en una misma orden.
- Cambiar la ubicacion de la seleccion de sucursal dentro del flujo.
- Revisar el orden de las pantallas antes de avanzar al prototipo medio nivel.
